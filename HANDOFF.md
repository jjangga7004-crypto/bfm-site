# BfM 사이트 — 디자이너 인수인계 가이드

안녕하세요! BfM 사이트 디자인을 이어서 맡아주실 분을 위한 안내입니다.
이 프로젝트는 **Claude Code(AI 코딩 도구)와 함께 작업**하도록 정리되어 있습니다.
코드를 직접 안 만지셔도, Claude에게 한국어로 요청하면 됩니다.

---

## 1. 무엇을 받게 되나
이 GitHub 저장소 전체입니다. 핵심 파일은 5개뿐:

| 파일 | 역할 |
|---|---|
| `index.html`    | PC 메인 랜딩 페이지 |
| `m.html`        | 모바일 랜딩 페이지 |
| `diagnose.html` | AI 피부진단 (카메라 + 얼굴인식) |
| `survey.html`   | 피부 진단 설문 |
| `result.html`   | 맞춤 처방 결과 |

빌드 과정이 없어서, **파일을 브라우저로 더블클릭하면 바로 보입니다.**

---

## 2. 시작하는 법
1. 이 저장소를 받으세요 (GitHub 접근 권한을 받았다면 그대로, 아니면 ZIP).
2. 이 폴더에서 **Claude Code를 켭니다.**
3. Claude가 같은 폴더의 `CLAUDE.md`를 자동으로 읽어 프로젝트 맥락을 이미 알고 있습니다.
4. 아래 예시처럼 한국어로 요청하면 됩니다.

---

## 3. Claude에게 이렇게 치세요 (예시 문장)

### 처음 파악할 때
```
이 프로젝트 구조랑 각 페이지가 뭐 하는 건지 설명해줘.
diagnose.html이 어떻게 동작하는지도 알려줘.
```

### 색/폰트 같은 디자인 토큰 바꿀 때 (⚠️ 5개 파일 다 바꿔야 함)
```
메인 강조색(--accent)을 #5B8DEF에서 (원하는 색)으로 바꿔줘.
5개 HTML 파일에 색이 각각 들어있으니 전부 일관되게 수정해줘.
```

### 특정 페이지만 손볼 때
```
index.html 히어로 섹션 여백을 더 넓히고 제목을 더 크게 키워줘.
바꾸고 나서 어떻게 보이는지 확인해줘.
```

### 모바일
```
모바일 랜딩(m.html)에서 버튼이 너무 작아 보여. 터치하기 좋게 키워줘.
```

### 확인/검토
```
방금 바꾼 거 브라우저에서 열어서 깨진 데 없는지 확인해줘.
```

> 팁: 요청은 **"어느 페이지에서, 무엇을, 어떻게"** 를 한 줄에 담으면 가장 잘 먹힙니다.
> 색·간격은 정확한 값(예: `#3E6FE0`, `24px`)을 주면 더 정확합니다.

---

## 4. 꼭 지켜주세요
- **디자인 토큰(색·폰트·반경)은 5개 파일에 따로 들어 있습니다.** 한 파일만
  고치면 페이지마다 달라지니, 항상 "5개 전부 일관되게"라고 요청하세요.
  (자세한 변수 목록은 `CLAUDE.md` 참고)
- **얼굴인식은 브라우저 안에서만** 돌아갑니다(사진이 서버로 안 나감 = 개인정보 보호).
  이미지를 어디로 전송하는 변경은 넣지 마세요.
- 변경 후엔 항상 해당 페이지를 브라우저로 열어 확인.
